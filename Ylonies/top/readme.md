# top

В этой задаче вам предлагается реализовать упрощенный аналог утилиты `top(1)`. Она должна выводить на экран информацию о работающих в данный момент процессах в порядке убывания использования CPU этими процессами. Эта информация выводится в виде таблицы, которая обновляется раз в секунду.

Пример того, как может выглядеть строчка с информацией о конкретном процессе:
```
    PID USER      PR  NI    VIRT    RES    SHR S  %CPU  %MEM     TIME+ COMMAND              
   4126 mesyarik  20   0 6495728 850084 252568 S   5,9   5,3 149:58.43 zoom                
```
Здесь:
- `PID` - идентификатор процесса;
- `USER` - имя юзера, от чьего имени данный процесс выполняется;
- `PR` - приоритет процесса;
- `NI` - niceness процесса;
- `VIRT` - количество виртуальной памяти, выделенной процессу (в килобайтах);
- `RES` - количество реальной физической памяти, занятой процессом;
- `S` - состояние процесса;
- `%CPU` - уровень загрузки процессора данным процессом (может быть больше $100$ на многоядерных процессорах);
- `%MEM` - процент физической памяти, занятой данным процессом;
- `TIME+` - время, прошедшее с момента старта процесса;
- `COMMAND` - название команды, запустившей процесс.

Столбец `SHR` можно не выводить.
Подробнее о значении чисел в столбцах можно прочитать документацию `top(1)`.

Большая часть вышеперечисленной информации о процессе берется из файлов `/proc/[pid]/stat`. Прочитайте документацию `proc(5)`, чтобы выяснить, в каком формате там хранится эта информация.

Список работающих в данный момент процессов также берется из виртуальной директории `/proc`. По сути, ваша программа (как и стандартный `top`) должна обойти все поддиректории `/proc`, соответствующие процессам, прочитать из них информацию и оформить ее в виде таблицы.

Чтобы узнать процент использования процессом физической памяти, нужно поделить `RES` на общее количество доступной в системе физической памяти.

Процент использования процессом CPU указывается за время, прошедшее с момента последнего обновления таблицы с информацией. Вам нужно из файла `/proc/[pid]/stat` узнать, сколько процессорного времени было использовано процессом за последнюю секунду (то есть с момента, как вы в прошлый раз обновляли информацию). Исходя из этих данных, посчитать, на сколько процентов CPU был загружен данным процессом.

В этой задаче можно использовать удобный вам язык программирования (необязательно Си или C++, можно Python, например). Однако вызывать из кода другие программы, которые за вас решают данную задачу, нельзя.
