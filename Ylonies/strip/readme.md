# Strip

Вам предлагается написать свой простой аналог программы `strip` из Linux.
Эта программа принимает на вход ELF-файл и вырезает из него секции, которые не являются необходимыми для работоспособности файла. Для простоты будем считать, что нас интересует лишь удаление секций `.symtab` (таблицы символов) и `.debug_*` (любых секций с отладочной информацией).

Ваша программа `strip` должна принимать на вход ELF-файл и преобразовывать его в другой ELF-файл, в котором по сравнению с исходным отсутствуют секции .symtab и все секции с отладочной информацией.

Ваша программа должна поддерживать флаг `-d`. Если этот флаг указан, то `.symtab` нужно сохранить, а удалить лишь секции с отладочной информацией.

Если исходный ELF-файл был исполняемым файлом или динамической библиотекой, то и файл после выполнения вашей версии strip над ним должен быть работоспособным исполняемым файлом или библиотекой, соответственно.

Пример использования: `./strip a.out`

Писать нужно на языке Си (можно использовать элементы C++). Для парсинга ELF-файлов можно использовать библиотеки `libelf` и `elfutils`. Можно установить их командой `sudo apt install libelf-dev elfutils`. Самостоятельно разберитесь, какие функции из данных библиотек помогут вам решить задачу.

Если на вход подан некорректный ELF-файл, то ваша программа должна корректно завершаться и сообщать об ошибке.

Делать ваш код переносимым на другие архитектуры процессора и версии операционной системы не нужно. Достаточно, чтобы он работал корректно на вашей версии.

Вызывать из кода стандартные программы вроде настоящего `strip` или `objcopy`, чтобы свести задачу к выполнению этих команд с нужными параметрами, нельзя. За это будет бан.
